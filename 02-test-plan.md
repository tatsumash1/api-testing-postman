# Тест-план

## Цель

Описать общий подход к тестированию API ReqRes.

## Объект тестирования

REST API сервиса ReqRes.

## Что тестируем (In-scope)

- получение списка пользователей
- получение одного пользователя
- создание пользователя
- обновление пользователя
- удаление пользователя
- регистрацию
- логин
- негативные сценарии

## Что не тестируем (Out-of-scope)

- работу базы данных на стороне сервиса
- внутреннюю бизнес-логику сервера
- производительность под высокой нагрузкой
- безопасность на глубоком уровне

## Инструменты

- Postman
- ReqRes Classic API
- GitHub для хранения тестовой документации

## Типы тестирования

- функциональное тестирование
- позитивное тестирование
- негативное тестирование
- проверка структуры ответа
- проверка статус-кодов
- проверка базовой валидации

## Критерии начала тестирования

- доступен сервис ReqRes с документацией
- есть список эндпоинтов
- доступен тестируемый API
- создана Postman Collection
- настроено окружение в Postman с переменными `base_url` и `api_key`

## Критерии окончания тестирования

- выполнены основные тест-кейсы
- зафиксированы найденные дефекты
- подготовлен тестовый отчет

## Риски

- публичный API может быть временно недоступен
- поведение тестового сервиса может отличаться от реального production API
- часть запросов может возвращать mock-ответы
