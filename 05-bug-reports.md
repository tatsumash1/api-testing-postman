# Баг-репорты

## Предисловие

В ходе тестирования ReqRes Classic API критичных дефектов обнаружено не было. Так как сервис является публичным учебным API и часть его поведения заранее задана документацией, реальные баги могут не проявляться при базовой проверке.

Эти баг-репорты не утверждают, что ReqRes действительно работает неправильно. Они используются как учебные примеры для портфолио.

## BUG-API-001. Успешная регистрация при некорректном формате email

| Поле | Значение |
| --- | --- |
| ID | BUG-API-001 |
| Заголовок | POST `/api/register` возвращает успешный ответ при некорректном формате email |
| Окружение | ReqRes Classic API |
| Endpoint | `/api/register` |
| Метод | POST |
| Severity | Major |
| Priority | High |
| Предусловия | В Postman выбрано окружение `ReqRes Dev`; заданы переменные `base_url` и `api_key`; в headers добавлен `x-api-key: {{api_key}}` |
| Шаги воспроизведения | 1. Открыть Postman. <br> 2. Отправить POST-запрос на `{{base_url}}/api/register`. <br> 3. Передать request body с некорректным email: `{"email":"eve.reqres.in","password":"pistol"}`. <br> 4. Проверить status code и тело ответа. |
| Ожидаемый результат | API возвращает `400 Bad Request` и сообщение об ошибке валидации email. |
| Фактический результат | API возвращает успешный ответ, несмотря на некорректный формат email. |
| Статус | Open |

## BUG-API-002. Avatar URL возвращается без информации о доступности изображения

| Поле | Значение |
| --- | --- |
| ID | BUG-API-002 |
| Заголовок | GET `/api/users/2` возвращает avatar URL без подтверждения доступности изображения |
| Окружение | ReqRes Classic API |
| Endpoint | `/api/users/2` |
| Метод | GET |
| Severity | Minor |
| Priority | Medium |
| Предусловия | В Postman выбрано окружение `ReqRes Dev`; заданы переменные `base_url` и `api_key`; в headers добавлен `x-api-key: {{api_key}}` |
| Шаги воспроизведения | 1. Открыть Postman. <br> 2. Отправить GET-запрос на `{{base_url}}/api/users/2`. <br> 3. Проверить поле `data.avatar` в теле ответа. <br> 4. Открыть URL изображения отдельно или отправить GET-запрос на avatar URL. |
| Ожидаемый результат | Avatar URL доступен и возвращает изображение с успешным status code. |
| Фактический результат | API возвращает avatar URL, но доступность связанного изображения не подтверждается в рамках ответа пользователя. |
| Статус | Open |

## BUG-API-003. Удаление несуществующего пользователя возвращает 204 No Content

| Поле | Значение |
| --- | --- |
| ID | BUG-API-003 |
| Заголовок | DELETE `/api/users/999` возвращает `204 No Content` для несуществующего пользователя |
| Окружение | ReqRes Classic API |
| Endpoint | `/api/users/999` |
| Метод | DELETE |
| Severity | Minor |
| Priority | Low |
| Предусловия | В Postman выбрано окружение `ReqRes Dev`; заданы переменные `base_url` и `api_key`; в headers добавлен `x-api-key: {{api_key}}` |
| Шаги воспроизведения | 1. Открыть Postman. <br> 2. Отправить DELETE-запрос на `{{base_url}}/api/users/999`. <br> 3. Проверить status code и тело ответа. |

## Примечание

Так как баги являются учебными, итоговый тестовый отчет должен отдельно указывать, что в ходе фактического тестирования критичных дефектов не найдено, а данные баг-репорты добавлены для демонстрации навыка оформления дефектов.
